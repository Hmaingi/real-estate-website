# Launch Admin Setup

The admin dashboard is available at `/dashboard`. If the admin is not logged in, it redirects to `/admin/login`.

## Environment Variables

Add these in your hosting provider, for example Vercel project settings:

```env
ADMIN_PASSWORD=choose-a-strong-private-password
ADMIN_SESSION_SECRET=choose-a-long-random-secret
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_SERVICE_ROLE_KEY=your-supabase-service-role-key
SUPABASE_STORAGE_BUCKET=property-images
```

Only the admin should know `ADMIN_PASSWORD`. Do not put the service role key in public frontend code.

## Supabase Table

Run this SQL in Supabase SQL Editor:

```sql
create table if not exists properties (
  id bigint generated by default as identity primary key,
  title text not null,
  location text not null,
  price text not null,
  numeric_price numeric not null default 0,
  type text not null check (type in ('Rent', 'Sale', 'Rent-to-own', 'Auction')),
  bedrooms integer not null default 0,
  bathrooms integer not null default 0,
  area text not null,
  images jsonb not null default '[]'::jsonb,
  description text not null,
  created_at timestamptz not null default now()
);
```

## Supabase Storage

Create a public storage bucket named `property-images`.

The dashboard uploads image files into that bucket and stores their public URLs on each property.

## How Admin Login Works

1. Open `/dashboard`.
2. The site redirects to `/admin/login`.
3. Enter the password from `ADMIN_PASSWORD`.
4. Add, edit, delete, and upload property images.
5. Click `Log Out` when done.
